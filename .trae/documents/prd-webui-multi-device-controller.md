# WebUI：多设备远控挂机控制台（PRD）

## 1. 背景与目标

本项目当前通过命令行脚本对单台设备执行“远控连接 → 微量键鼠操作 → 断开连接”的防掉线循环。随着设备数量增加，需要一个可视化控制台来集中管理多台设备的连接信息、运行状态、定时运行时长，并在每次执行动作后留存远控画面截图用于回溯。

**目标：**
- 提供一个 WebUI，用于配置与管理多台设备（ID/密码 + 独立配置项）
- 支持对每台设备单独启动/停止，并在启动前输入“运行倒计时 N 小时 M 分”
- 每次执行防掉线动作后保存远控画面截图，每台设备仅保留最近 10 张

## 2. 用户画像

- 小白用户：希望“点几下就能跑起来”，不理解 Python/依赖/命令行细节
- 进阶用户：希望能编辑每台设备的独立参数、查看日志、快速排障

## 3. 使用场景（User Stories）

1. 作为用户，我想新增一台设备，填写设备名、远控 ID、密码，并设置独立的循环间隔、初始化等待等参数。
2. 作为用户，我想在设备列表看到每台设备的运行状态（未运行/运行中/已停止/异常退出）。
3. 作为用户，我想点击“开始”，并输入运行时长（N 小时 M 分），让脚本自动在到期后停止。
4. 作为用户，我想随时点击“停止”终止该设备的脚本进程。
5. 作为用户，我想在每台设备的详情页看到最近 10 张截图，按时间倒序排列。

## 4. 功能需求

### 4.1 设备配置管理

- 设备字段（必填）
  - 设备名称（用于 UI 展示）
  - 远控 ID
  - 远控密码
- 设备字段（可选）
  - 独立配置项（覆盖 `config.yaml` 的部分字段，如循环间隔、键鼠随机范围等）
  - 备注
- 支持新增/编辑/删除设备
- 设备配置落盘保存，重启 Web 服务后仍可恢复

### 4.2 启动/停止

- 每个设备提供：
  - “开始”按钮
    - 弹窗/抽屉输入运行时长：小时（0-999）+ 分钟（0-59）
    - 支持快捷值：6h5m（默认）、1h、2h、8h
    - 确认后启动该设备的脚本，并将本次运行时长写入该设备的运行配置（覆盖 `run.auto_exit_after_seconds`）
  - “停止”按钮
    - 结束该设备的脚本进程（尽量优雅停止；必要时强杀）
- 状态显示：
  - 未运行
  - 运行中（显示启动时间、预计结束时间/剩余时间）
  - 已停止（显示停止时间/退出码）
  - 异常（显示错误摘要）

### 4.3 截图留存

- 每次执行“防掉线动作（一次 safety_movement）”后保存一张截图
- 存储策略：
  - 每台设备单独目录
  - 文件名包含时间戳
  - 每台设备最多保留最近 10 张（超出则删除最旧）
- WebUI 展示：
  - 设备详情页显示最近截图列表（时间 + 缩略图 + 点击查看大图）

### 4.4 配置项策略

- 仓库根目录 `config.yaml` 作为全局默认
- 设备可持有“覆盖配置”（YAML 片段或结构化表单）
- 实际启动时：全局默认配置与设备覆盖配置进行合并，生成“设备运行配置”

## 5. 非功能需求

- 可用性：单文件一键启动（`python -m uvicorn ...`），无需前后端分离部署
- 安全性：
  - UI 默认仅监听 `127.0.0.1`
  - 明确提示：设备密码属于敏感信息；存储需限制文件权限
- 可维护性：
  - 配置与状态数据采用清晰的目录结构与 JSON/YAML 格式
  - 日志落盘便于排障

## 6. 边界与约束

- MVP 优先支持 Linux RustDesk 场景（因为脚本在 Linux 无头环境更常见）
- Windows UU 场景可作为后续扩展（同一 UI 框架下增加 runner 类型）
- 截图能力依赖运行环境：
  - Linux 优先使用系统截图工具（如 `scrot`）或其它可用方案

## 7. 交互与页面（桌面优先）

### 页面 1：设备列表
- 表格/卡片：
  - 设备名、ID（部分脱敏）、状态、剩余时间
  - 操作：开始、停止、编辑、详情

### 页面 2：设备详情
- 运行信息（启动时间/预计结束/最近错误）
- 最近 10 张截图
- 最近日志摘要（可选）

### 页面 3：设备编辑
- 基础信息（名称/ID/密码/备注）
- 覆盖配置（先做文本 YAML 编辑器，后续可做表单化）

## 8. 验收标准

- 能新增至少 2 台设备并保存配置
- 每台设备都能独立启动/停止，启动时支持输入 N 小时 M 分
- 运行期间能持续产生截图并在 UI 可查看，每台设备最多保留 10 张
- Web 服务重启后，设备列表与截图仍可正常查看
